## 🔐 ส่วนที่ 1 — Hardening ความปลอดภัย

### 1.1 เปลี่ยนการส่งเข้า Google Sheet เป็นลายเซ็น HMAC
- เดิม: ส่ง "รหัสลับ" ตรง ๆ ผ่านเน็ต → ถ้าถูกดักจับ = รหัสหลุด
- ใหม่: ส่งแค่ **ลายเซ็น HMAC-SHA256** ที่คำนวณจากรหัส (รหัสไม่วิ่งผ่านเน็ต) + ผูก timestamp (กันดักแล้วยิงซ้ำ) + เซ็นทั้ง payload (กันแก้ยอดกลางทาง)
- **ทดสอบเจาะ 8 เคสผ่านครบ:** ลายเซ็นมั่ว/คีย์ผิด/replay/แก้ยอด/รหัสเก่าที่เคยหลุด → ปฏิเสธหมด

### 1.2 ปิดหน้า/API ที่เปิดโล่ง
- `quotes`, `quotes/[id]`, `ebook-stats` — เดิมเปิดด้วย "URL ลับ" อย่างเดียว ใครได้ลิงก์ก็เข้าได้ → **ปิดด้วยรหัส owner**
- `POST /api/quotes/[id]/update` — เดิมกันด้วย header ที่ปลอมได้ → ต้องมี cookie owner
- `POST /api/quotes/[id]/analyze` — **เดิมไม่ตรวจอะไรเลย** ใครก็ยิงให้เผาเงินค่า AI ได้ → ปิดด้วยโทเคนภายใน

### 1.3 Rate limiting (กันใช้เกิน/กันเดารหัส)
- อ่านเอกสาร: 40 ใบ/ชม. · 150 ใบ/วัน (กันค่า API บานปลาย)
- อัปโหลด: 120 ไฟล์/ชม.
- ใส่รหัสผิดถี่ → **หน่วงเวลา** (ไม่บล็อก เพื่อไม่ให้ถูกกลั่นแกล้งจนเจ้าของเข้าไม่ได้)

### 1.4 แก้บั๊ก (เจอตอนใช้จริง)
- **ลายเซ็นพังเมื่อเอกสารมีภาษาไทย** — ตัว Apps Script แปลงตัวอักษรไทยเป็นไบต์คนละแบบกับระบบ → แก้โดยเซ็นบน base64 (ทั้งสองฝั่งเห็นเหมือนกัน) · เจอเพราะทดสอบกับบิลจริง
- **ลองใหม่ครบ 3 ครั้งแล้วใบค้างถาวร** — แยกเพดาน: อ่านใหม่ 3 ครั้ง (เสียเงิน) / ส่ง Sheet ใหม่ 20 ครั้ง (ไม่เสียเงิน)
- เพิ่มหน้าเช็ก env (owner-only) — เวลาตั้งค่าผิดจะไล่หาสาเหตุได้เร็ว ไม่ต้องเดา

## 📦 ส่วนที่ 2 — Profile A "รับเข้าสต็อก" (ของใหม่ทั้งหมด)

**Pain ที่แก้:** เจ้าของต้องคีย์อะไหล่เข้าสต็อกทีละชิ้น + ต้องกด "Sync สต็อก" เอง (ไม่สะดวก)

**Flow ใหม่:**
```
📄 ถ่ายใบส่งของ → 🤖 AI แตกรายการ → เติมอัตโนมัติ (SKU/หมวดหมู่/สภาพ)
              → 👤 ตรวจ + เติมราคา/ที่เก็บ → ✓ เข้าสต็อก (ไม่ต้อง Sync)
```

### พิสูจน์ครบวงจรกับใบจริง
ใบส่งของ 39,000 บาท (3 รายการ ลายมือ):

| # | AI อ่าน | ตัวเลข |
|---|---------|--------|
| 1 | กระจังหน้า W140 · 3 ชิ้น | ✅ 3×5,000=15,000 |
| 2 | ไฟท้าย W140 · 4 ชิ้น | ✅ 4×3,000=12,000 |
| 3 | ไฟแต่งหน้า W140 · 4 ชิ้น | ✅ 4×3,000=12,000 |

**ตัวเลขถูก 9/9** · ชื่อลายมือถูก 2/3 (บรรทัดที่พลาด ระบบให้ความมั่นใจต่ำ = เตือนถูกจุด) · เข้า Ledger ครบทั้ง 3 ตัว **โดยไม่ต้องกด Sync**

### ฟีเจอร์เด่น
- **auto SKU** — ต่อเลขจากตัวล่าสุดจริงของรุ่นนั้น (W140 ล่าสุด 140-031 → ใหม่ 140-032/033/034) · กันเลขชนกับใบอื่นที่ยังไม่ยืนยัน
- **auto หมวดหมู่** — จับคู่จากชื่อ ("กระจังหน้า" → "BDY-05 กระจังหน้า") ~25 หมวดจากชีตจริง · ไม่ชัด = เว้นว่าง ไม่มั่ว
- **auto สภาพ** — ตั้ง "มือสอง-A" ให้ (แก้ได้)
- **เชื่อถือตัวเลขจากคณิตศาสตร์** — บวกคูณลงตัว = อ่านถูก · ชื่อลายมือต้องผ่านตาเจ้าของเสมอ (ห้ามเข้าสต็อกอัตโนมัติ)
- ต้นทุน ~1.3 บาท/ใบ

### หลักการสำคัญที่ตัดสินใจ
เขียนเข้า `stock_records` ตรง (แทนการเขียนชีตแล้วให้เจ้าของ Sync เอง) — เพราะ **ตัวปัญหาคือขั้น Sync ไม่ใช่เรื่องชีต** การเขียนตรงจึงข้าม Sync ทั้งขั้น หมวดหมู่/สภาพ เก็บในช่องหมายเหตุ (เพราะ stock_records ไม่มีคอลัมน์นี้)

## 🧹 ส่วนที่ 3 — เก็บงาน

**รวมทางลงขายให้เหลือที่เดียว** — เดิมลงขายได้ 2 ที่ (Ledger + เมนู "ขายออก") เขียนตารางเดียวกัน = เสี่ยงลงซ้ำ นับเบิ้ล
- เก็บ **"ขายออก"** เป็นที่ลงขาย (เก่งกว่า: มีจำนวน + ราคาอัตโนมัติ + ตัดสต็อก)
- Ledger เหลือหน้าที่ **ดู/แก้** (ซ่อนฟอร์มลงขายใหม่ · ไม่ลบข้อมูล)
